# Impeccable — Design Context

## Design Context

### Users
Fujifilm camera owners (X-series) who shoot with in-camera "film simulation recipes."
They manage the **7 physical custom slots (C1–C7)** on their body and a larger personal
collection of recipes. Primary context: at a desk or in the field, phone connected to the
camera over **USB-C OTG (PTP / USB RAW CONV. mode)**. Job to be done: read what's loaded in
each C-slot, edit recipes, write them back to the camera, and keep an organized phone library
of recipes to swap in. Audience skews enthusiast/pro photographer — comfortable with terms
like Dynamic Range, Grain Effect, Color Chrome, WB Shift, Highlight/Shadow Tone, Clarity.

### Brand Personality
Three words: **premium, editorial, focused.** Quiet confidence — a precision tool for
photographers, not a toy. Voice is terse and technical but human. Should evoke the feeling of
a darkroom / a well-made light meter: calm, deliberate, content-forward. No clutter, no
gimmicks, no marketing fluff.

### Aesthetic Direction
- **Theme:** dark only (v1). Near-black warm background (`~#0D0D0D`), panels lifted by a
  slightly lighter warm-gray tone rather than drop shadows.
- **Accent:** a single disciplined **warm gold / brass** (`~#C99A4E`). One accent does all the
  work — wordmark, active nav, icons, primary actions, links. Never introduce a second accent
  color. (User is happy that this reads "Claude Code"–like; it's the intended house style.)
- **Typography:** large **bold UPPERCASE** recipe names; small letter-spaced uppercase section
  labels (e.g. `EFFECTS`, `TONE`); mixed-weight wordmark (bold + light). Clean sans throughout.
  Values right-aligned, bold, white.
- **Components:** dark pill chips for at-a-glance recipe summaries (`DR100%`, `CC WEAK`,
  `HL +1`); label-left / value-right rows with a small gold line-icon per property; sticky
  bottom CTA for the primary action on a screen.
- **Reference (liked, not to be copied):** `com.bechir.fujifilmrecipes` (screenshots in
  `references/`). We adopt its warm-gold-on-dark *look* but deliberately diverge on
  *page structure*.
- **Anti-references:** generic Material-3 light/purple defaults; busy multi-color dashboards;
  skeuomorphic camera-dial kitsch; the throwaway Codex "test bench" UI currently in the repo
  (to be scrapped entirely — not a design constraint).

### Differentiation (structure, not color)
We earn our own identity through **page structure**, since the palette intentionally resembles
the reference. Chosen structure: **Slot-board first.** The home screen (`MY CAMERA`) is the
camera's **C1–C7 slots as a board** showing what's loaded right now — the reference app hides
the slots; we make them the spine. Navigation is **3 tabs**: `CAMERA · LIBRARY · PROFILE`.
Core screens: Slot Board (offline connect-guide + connected states) → Slot Detail → Recipe
Editor (shared) → Library → Recipe Detail → Profile/Settings. Three core flows: read
(connect → board populates → tap slot), write/edit (slot → edit → write to camera), manage
library (create/edit/duplicate/delete; "Save to Library" from any slot).

### Tech & Workflow Notes
- **Platform:** native Android, **Jetpack Compose + Material 3**, package `com.paeki.fujirecipes`.
- **Design loop:** iterate via Compose `@Preview` (+ Live Edit on emulator). For remote review,
  prototype look in HTML and/or render real Compose screens to PNG (Paparazzi-style) and push to
  the user's Google Drive, since the user often works via remote control and cannot see the
  desktop screen or run the emulator directly.

### Design Principles
1. **The camera is the spine.** Always orient the UI around the 7 physical C-slots and the
   read → edit → write loop, not around a generic library.
2. **One accent, used sparingly.** Warm gold on warm-black; let weight, size, and spacing —
   not color — create hierarchy.
3. **Content over chrome.** Big confident typography, generous dark space, minimal borders;
   the recipe data is the hero.
4. **State honesty.** Always show connection state and whether what's on screen reflects the
   camera, the phone library, or unsaved edits.
5. **Diverge by structure, not decoration.** When tempted to copy the reference app, change the
   layout/flow instead — keep the aesthetic, make the structure ours.
