# Fujifilm USB Recipe Manager — Project Brief

## What we're building
An Android app that talks to a Fujifilm X-H2 (and other X-series bodies) over
USB OTG and lets the user read, write, and manage individual film-simulation
recipes stored in the camera's custom shooting slots (C1–C7) — no PC, no
X Acquire.

## How the camera must be set up
MENU → SET UP → USB Setting → **USB RAW Conv. / Backup Restore**
Physical connection: USB-C OTG cable, camera as peripheral, Android phone as host.

## Stack
- Kotlin + Jetpack Compose
- Min SDK 26 (Android 8)
- `android.hardware.usb` — no root, no NDK
- No external deps for the core protocol

## Protocol
The wire protocol is documented in **PROTOCOL.md** (root of this repo).
That document is the source of truth — it was reverse-engineered from
`reverse-engineering/Fuji Recipe.apk` and is kept in sync as we build the app.

Short version: it's standard PTP over USB with Fujifilm vendor device
properties in the `0xD18E–0xD1A5` range. Recipes are read/written one PTP
device property at a time after first writing the C-slot index to the
slot-selector property `0xD18C`. See PROTOCOL.md for the full opcode table,
byte layouts, enum values, and end-to-end session walkthrough.
